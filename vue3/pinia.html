# 使用

在setup组件当中使用,需要Vue当中显式引入组件

import {createPinia} from "pinia"
import {createApp} from "vue"

const app = createApp({});
app.use(createPinia());

# 定义store

store是pinia的核心概念，所有的state、getters、action都在store里面定义 这里的state就是我们常用的一些状态变量，类比组件当中的data，但是这里的state是可以跨组件使用的 定义

在组件中使用store之前，如何定义store呢？Pinia 使用defineStore函数创建一个useStore函数来返回store的实例对象

import { defineStore } from "pinia";

export const useStore = defineStore("main", {
    state: ()=>{
        return {
            name: "bjorn"
        }
    },
 })

# state

Pinia 可以直接修改、访问state,也可以通过$patch批量修改访问state 值得一提的使用，可以在 Vue devtools 里面看到 Pinia 的state值,方便进行调试和开发

  • 直接访问
store.name = "orz";
1
  • 通过$patch访问 $patch可以传递一个函数和对象，如果传递对象，每次更改都会重新创建一个对象，代价比较高昂
store.$patch(state => {
    state.name = "lxxorz"
})
1
2
3
  • 订阅更新

$subscribe会监听state的变化，然后执行订阅的函数,其中第二个参数用于配置当组件销毁之后，订阅是否还存在

store.$subscribe(callback, {detached: true});
